================================================================================
KNOWN LIMITATIONS & DESIGN DECISIONS
================================================================================

BY DESIGN (Intentional):
───────────────────────────────────────────────────────────────────────────────
1. No interactive password pre-configuration
   - Prevents hardcoded credentials
   - User must enter password interactively
   - Design rationale: Security best practice

2. Hostname validation only RFC 952 compatible
   - No support for uppercase in hostnames (DNS standard)
   - Prevents injection attacks
   - Design rationale: Security & compatibility

3. No SSH key management
   - Out of scope for this tool
   - Separate security process needed
   - Design rationale: Single Responsibility Principle

4. No automatic firewall configuration
   - Differs by distribution and use case
   - User responsibility
   - Design rationale: Operator-controlled policies
